About Flood-Iction

This page provides a brief summary of how Flood-iction works.

Flood Prediction Model

Flood-iction is built upon the concept of a model. At the moment this model is only capable of one form of analysis - when the Brisbane River overflows.
When given a location, it finds the height of this point using (reference????) and then provides what frequency a flood event that can flood this area would occur in.
For example, a location that is just 0.5m above the Brisbane River and adjacent to it would flood more frequently than one which is 20m above the river and 1km away.
In the future we plan to update this to include other kinds of analysis and give a weighted flood risk score considering these factors. For example:
  • Flooding due to releases from the Wivenhoe Dam
  • Flooding due to rainfall in various amounts and time periods
  • Flooding due to tidal changes such as King Tides

Data Visualization

This website was developed to visualize the data generated by the model for the user. It runs on Flask, a bare bones Python web application. The style of the pages is mostly done using Bootstrap


After the user provides an address or location to search, the result is displayed on a map. This map is created by using a library called Folium, which creates a Leaflet map.
On top of this map is a Contour map, typically used to notate changes in terrain height. Here it is not height being displayed, but instead flood risk.
The model is able to at what frequency a flood severe enough to cause the given water height would occur at. This is fit on a linear scale to give the risk shown on the map.


Some of this data is also given to the user in table form for various flood height levels.

GeoLocation

To provide specific flood information about a given location, Flood-iction needs to find where that is on a map.
Luckily there is an existing Python library that does just this. The GeoPy library contains dozens of GeoCoders - APIs that turn an address "e.g. 123 John Street" or a description "e.g. Brisbane Children's Hospital" into map co-ordinates.
In particular, Flood-iction uses Nominatim. Nominatim uses open source data from the OpenStreetMap Project to produce co-ordinates.
This open source data is not perfect - but in and around the Brisbane City area, most of the data is complete and accurate.

In no uncertain terms: Flood-iction does not actively seek out any user data that isn't supplied, such as by attempting to locate the user by their IP Address. Any addresses or locations that Flood-iction is provided by the user are not stored persistently, nor is it provided to any third parties.
